# Full Version Outlook

## Public demo versus private full version

The public repository is intentionally limited to the signer workflow and to the materials that are safe and useful to present publicly.

The private full version is expected to expand beyond the public signer demo in areas such as:

- broader UI polish and packaging
- wider operational workflows
- larger private API and CLI coverage
- stronger release engineering and distribution paths
- extended security controls and project policies
- richer operational audit and lifecycle tooling
- possible future expansion toward other NIST PQ standards

## Longer-term cryptographic direction

The current public focus is **FIPS 204 / ML-DSA**.

The private roadmap may later expand into:

- **FIPS 205 / SLH-DSA** as an additional signature track
- **FIPS 203 / ML-KEM** for secure exchange / KEM-oriented workflows

## What is intentionally not public here

The public demo does not expose the full private implementation surface, the full private operational logic, or the full long-term product scope.
